Chapter 1. Why We Program

A program is a set of step-by-step instructions you give the computer. Python is the language you write those instructions in. The computer follows them in order, from the top of the file to the bottom.

Words to know

  • Program. A list of instructions.
  • Python. The language you write them in.
  • print. The command that shows text on the screen.
  • Comment. A note for humans, starting with the hash mark. Python ignores it.

Example

# This is a comment
print("Hello, World")
print("I am learning to code")

Chapter 2. Variables and Expressions

A variable is a labeled box that holds a value so you can use it later. It can hold text or numbers, and you can do math with numbers. The input command lets the program ask you a question and store your answer.

Words to know

  • Variable. A named box that holds a value.
  • String. Text, always inside quotes.
  • Integer. A whole number.
  • Float. A number with a decimal point.
  • input. Asks a question and stores what you type. It always gives back text.
  • int and float. Convert text into a number so you can do math with it.

Example

name = input("What is your name? ")
print("Hello", name)

age = int(input("How old are you? "))
print("Next year you will be", age + 1)

Chapter 3. Conditional Code

Sometimes you want the program to make a choice. The word if checks whether something is true. If it is true, the indented lines below it run. The word elif checks another possibility, and else covers everything left over. Indentation, meaning the spaces at the start of a line, is how Python knows which lines belong together.

Words to know

  • if. Runs code only when something is true.
  • elif. Checks another condition if the first was false.
  • else. Runs when nothing above it was true.
  • try and except. Attempts risky code, and catches the error instead of crashing.

Example

hours = 45
if hours > 40:
    print("You worked overtime")
elif hours == 40:
    print("Exactly full time")
else:
    print("Under full time")

Chapter 4. Functions

A function is a named block of code you can reuse. You define it once with the word def, then call it whenever you need it. A function can take an input, called a parameter, and can hand back a result with the word return.

Words to know

  • def. Defines a new function.
  • Parameter. An input the function receives.
  • return. Hands a result back to whoever called it.
  • Call. Running the function by its name.

Example

def greet(person):
    print("Hello", person)

def double(number):
    return number * 2

greet("Robert")
print(double(21))

Chapter 5. Loops and Iteration

A loop repeats work so you do not have to write it many times. A for loop walks through a set of items one at a time. A while loop keeps going until something changes. Common patterns are keeping a running total, counting things, and remembering the largest value you have seen so far.

Words to know

  • for. Repeats once for each item.
  • while. Repeats until a condition changes.
  • range. Makes a series of numbers to loop over.
  • break. Leaves the loop early.
  • continue. Skips to the next time around.
  • None. A special empty value, useful for starting a largest or smallest search.

Example

total = 0
for number in [3, 8, 5]:
    total = total + number
print("Total is", total)

Chapter 1 Exercises

1.1 Three lines about you Easy

Print three separate lines: your name, your town, and one thing you want to build with code.

Use print three times, once per line. Each piece of text goes inside quotes.

print("Robert")
print("Boston")
print("A tool for my website")

1.2 Add a comment Easy

Write a program that prints one line, and put a comment above it explaining what the line does.

A comment starts with the hash mark. Python skips over it entirely.

# This line greets the user
print("Learning Python")

1.3 Break it on purpose Medium

Run a line with a missing quote mark and read the error Python gives you. Then fix it. Learning to read errors is a real skill.

Python will say something about an unterminated string. That means a quote mark is missing. Add the closing quote before the closing bracket.

Chapter 2 Exercises

2.1 Favorite things Easy

Make three variables for a color, a food, and a number. Print a sentence that uses all three.

You can list several things inside one print, separated by commas. Python puts a space between them.

color = "blue"
food = "pizza"
number = 7

print("I like", color, "and", food, "and the number", number)

2.2 Ask and answer Easy

Ask the person for their city, then print a sentence welcoming them from that city.

The input command hands back whatever was typed. Store it in a variable, then use that variable in your print.

city = input("What city are you from? ")
print("Welcome, friend from", city)

2.3 Text is not a number Medium

Ask for a number, then print that number doubled. Watch out, because input always gives you text, so you must convert it first.

Wrap the value in int to turn text into a whole number. If you skip this, Python will repeat the text instead of doing math.

answer = input("Give me a number: ")
number = int(answer)
print("Doubled, that is", number * 2)

2.4 Rectangle area Medium

Ask for a width and a height, then print the area. Use float so decimals work too.

width = float(input("Width: "))
height = float(input("Height: "))
area = width * height
print("The area is", area)

2.5 Seconds to minutes Challenge

Ask for a number of seconds. Print how many whole minutes that is and how many seconds are left over.

Two double slashes give you whole number division. The percent sign gives you the remainder.

seconds = int(input("How many seconds? "))
minutes = seconds // 60
left = seconds % 60
print(minutes, "minutes and", left, "seconds")

Chapter 3 Exercises

3.1 Hot or cold Easy

Ask for a temperature. Print one message if it is above 75, and a different message otherwise.

temp = int(input("Temperature: "))
if temp > 75:
    print("Warm out there")
else:
    print("Bring a jacket")

3.2 Three way sort Medium

Ask for a number. Say whether it is positive, negative, or exactly zero. You will need elif for this.

Check greater than zero first, then less than zero, then let else catch the only case left, which is zero.

number = int(input("Give me a number: "))
if number > 0:
    print("Positive")
elif number < 0:
    print("Negative")
else:
    print("Zero")

3.3 Letter grade Medium

Ask for a score out of 100. Print A for 90 and up, B for 80s, C for 70s, and F below that.

score = int(input("Score: "))
if score >= 90:
    print("A")
elif score >= 80:
    print("B")
elif score >= 70:
    print("C")
else:
    print("F")

3.4 Catch the bad input Challenge

Ask for a number, but do not let the program crash if the person types letters. Use try and except to print a friendly message instead.

Put the risky conversion inside try. If it fails, Python jumps down to except instead of stopping the program.

raw = input("Type a number: ")
try:
    number = int(raw)
    print("You gave me", number)
except:
    print("That was not a number")

Chapter 4 Exercises

4.1 Your first function Easy

Write a function called cheer that prints a short encouragement. Call it three times.

def cheer():
    print("Keep going, you are learning")

cheer()
cheer()
cheer()

4.2 Function with a parameter Easy

Write a function that takes a name and prints a greeting using it. Call it with two different names.

def greet(person):
    print("Good to see you,", person)

greet("Robert")
greet("Kristina")

4.3 Return a value Medium

Write a function that takes two numbers and returns their sum. Print the result of calling it. Notice the difference between print and return.

print shows something on the screen. return hands the value back so you can use it elsewhere. A function that only prints gives back nothing.

def add(a, b):
    return a + b

result = add(3, 4)
print("The sum is", result)

4.4 Temperature converter Challenge

Write a function that takes a Fahrenheit temperature and returns the Celsius value. Test it with 32, which should give 0.

Subtract 32 first, then multiply by 5, then divide by 9. Brackets control the order.

def to_celsius(f):
    return (f - 32) * 5 / 9

print(to_celsius(32))
print(to_celsius(212))

Chapter 5 Exercises

5.1 Count to ten Easy

Use a for loop with range to print the numbers 1 through 10.

range stops just before the second number, so use 11 to reach 10.

for number in range(1, 11):
    print(number)

5.2 Running total Medium

Add up all the numbers in a list and print the total. Start your total at zero before the loop begins.

numbers = [4, 9, 2, 7, 1]
total = 0

for n in numbers:
    total = total + n

print("Total:", total)

5.3 Count and average Medium

Loop through a list, keep both a total and a count, then print the average at the end.

Add to total each time, and add one to count each time. Divide at the very end, outside the loop.

numbers = [10, 20, 30, 40]
total = 0
count = 0

for n in numbers:
    total = total + n
    count = count + 1

print("Average:", total / count)

5.4 Skip and stop Medium

Loop through numbers 1 to 20. Skip every even number using continue, and stop the loop entirely when you reach 15 using break.

A number is even when dividing it by 2 leaves no remainder. The percent sign gives you the remainder.

for n in range(1, 21):
    if n > 15:
        break
    if n % 2 == 0:
        continue
    print(n)

5.5 Find the biggest Challenge

Walk through a list and find the largest value, without using the built in max command. Start your best value at None and replace it whenever you see something bigger.

Two cases to handle. If largest is still None, this is the first number, so take it. Otherwise, only replace it when the new number is bigger.

numbers = [12, 45, 3, 78, 22]
largest = None

for n in numbers:
    if largest is None or n > largest:
        largest = n

print("Largest:", largest)

5.6 Keep asking until done Challenge

Use a while loop to keep asking for numbers until the person types the word done. Add them up as you go, then print the total.

Check for the word done first and break out. Then use try and except so bad entries do not crash it.

total = 0

while True:
    entry = input("Number, or done to finish: ")
    if entry == "done":
        break
    try:
        total = total + int(entry)
    except:
        print("That was not a number, skipping")

print("Total:", total)

What Makes It Feel Hard

  • A new way of thinking. You break problems into small logical steps. This is different from how we usually think.
  • New words. Variable, function, loop. These are all new at first, and that is normal.
  • Mistakes show up right away. When something is wrong, the computer tells you. Learning to fix errors takes practice.
  • Repetition matters. You write the same kind of code many times before it feels natural.

Why You Have an Advantage

  • You break big challenges into steps. That is exactly what programming is.
  • You have time to practice. Consistency beats intensity every time.
  • You have a real reason to learn. You want to build tools for your website and your work. That keeps you going.
  • You understand persistence. You have faced challenges far harder than a syntax error. You know how to push through.

Remember This

Coding is not harder than learning to drive or learning an instrument. It just feels different at first. When something feels hard, it usually means you are learning. Every expert started exactly where you are. The only difference is that they kept going.
